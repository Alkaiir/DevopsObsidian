> Зачем мне **Linux CLI**, если я пользуюсь **Windows** или **MacOS**?

Это частый вопрос от тех, кому советуют изучить данный материал в перспективе становления разработчиком/QA ☝️

Причин несколько:

1. Некоторые утилиты (программы) работают только в консольном режиме - т.е., с использованием команд.
2. **Linux CLI** - это некий стандарт, многие его команды перекочевали и в **PowerShell** (который используется в **Windows**), а **MacOS** - это тот же **Linux** (тоже **unix** операционная система).
3. Сервера, как правило, не имеют графической оболочки, а потому единственный способ управления ими - при помощи команд.
4. **CI/CD** платформы используют образы **unix** операционных систем, и действия по "непрерывной интеграции и доставке" описываются, как раз таки, при помощи команд.

И это малая часть, давайте разберём эти примеры подробнее...

## Работа с утилитами (программами)

Думаю, о команде `ping` слышали многие - ею пользуются, когда хотят проверить "есть ли сеть".

Допустим, вы открываете браузер и пытаетесь открыть [https://google.com](https://google.com/), но ничего не выходит - страница почему-то не открывается. Возможно проблема в браузере? А есть ли сеть вообще?

На данном этапе вы можете открыть терминал и выполнить команду `ping google.com` и увидеть, есть ли подключение до этого ресурса.

Общение с **Git**-площадками с использованием [SSH-ключей](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent) давно является стандартом, и в некоторых командах на вас могут косо посмотреть, если вы, заявленный **Senior**, будете работать через **HTTPS** и хлопать глазами при вопросе "почему не SSH?" 🙂

Конечно, существуют и графические интерфейсы (и в каких-то случаях они довольно удобны), но для многих манипуляций быстрее и удобнее использовать именно командный интерфейс.

Если вы работаете (или планируете работать) с кодом - вы точно не пройдёте мимо **Git**, поэтому **(!!!) heads up (!!!)**, внимайте информацию до тех пор, пока не постигните дзен.

## Сервера

> Вроде не собираюсь быть сисадмином, причём тут вообще сервера?

Если вы планируете быть QA/разработчиком - вы с огромной вероятностью столкнётесь с серверами, а именно - с необходимостью попасть на них, "почитать логи", и чего-нибудь ещё...

Например, какое-то приложение умерло, и вас попросили почитать логи - посмотреть, с какой ошибкой оно упало. Так как задача серверов - "просто стоять включёнными" и отвечать на запросы пользователей, то им не нужна графическая оболочка -- зачем она, если физически рядом с этим устройством никто не находится, да и заходят на это устройство крайне редко? Да и тем более, под графическую оболочку ещё и дополнительные ресурсы нужны...

![](https://ucarecdn.com/c2eece16-9359-4a06-8a9b-7748c5ba702b/)

Намёк понятен? Без умения работать с терминалом тут точно не обойтись.

Таким образом, используя команды в терминале, вы можете как минимум:

1. Подключиться к серверу.
2. Прочитать логи.
3. Перезапустить сервер.

## CI/CD  [[CI-CD]]

> Хочу автоматизировать процессы

С этой мыслью и не только, начинающие IT-специалисты начинают задумываться о изучении **Linux CLI**. Стоит обратить внимание и на то, что опыт работы с **CI/CD** является практически обязательным для каждого инженера (QA/Developer).

Возьмём в пример любое веб-приложение - его разработчик пишет код у себя на компьютере, а потом заходит на сервер и добавляет этот новый код туда, в общий доступ.

А что если разработчик не один?

Что, если таких изменений много? - Это превращается в некую рутину, причём довольно длительную, если весь этот процесс выполнять вручную.

Здесь на помощь приходят платформы непрерывной интеграции, в которых можно полностью автоматизировать сборку и доставку приложения конечному пользователю, а именно:

1. **Проверить** код на качество - рабочий ли он вообще, и не используются ли в нём какие-либо антипаттерны, принятые на проекте.
2. **Собрать** приложение (build) - код необходимо "превратить" в работающее приложение, прежде чем его запускать.
3. **Протестировать** приложение (автотесты).
4. **Загрузить** приложение на сервер.

И описать эти действия в **CI/CD** процессе можно именно при помощи **unix** команд ☝️

Пример **CI/CD pipeline** с **UI** тестами на **Python** + **Selenium**:

```yaml
name: Test

on:
  pull_request:
    branches: [ main ]

jobs:
  configure:
    runs-on: ubuntu-latest
    outputs:
      containerUser: ${{ steps.get-user.outputs.containerUser }}
    
    steps:
      - id: get-user
        run: echo "::set-output name=containerUser::`id -u`:`id -g`"

  test:
    needs: configure
    runs-on: ubuntu-latest
    container:
      image: circleci/python:latest-browsers
      options: --user ${{ needs.configure.outputs.containerUser }}

    steps:
      - uses: actions/checkout@v2

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt

      - name: Run tests
        run: python -m pytest
```

И изучение **Linux CLI** в данном случае - это вершина айсберга. После прохождения данного курса у вас не будет навыков работы с **CI/CD**, но будет необходимый задаток, чтобы начать изучать этот процесс.

## Основные понятия

**CLI** (**C**ommand **L**ine **I**nterface) - оболочка, через которую выполняются команды, например:

- **PowerShell** или **Command Prompt (cmd)** на Windows,
- **zsh** на **MacOS**.

**Директория** (directory) - иными словами "папка" (folder), или "путь" (path).

**Регулярное выражение** (**reg**ular **ex**pression или regex) - текстовое выражение, которое позволяет отфильтровать/найти в тексте искомые значения.

Например, **email** адреса имеют паттерн `<пользователь>@<домен>`  - то есть, все они написаны по одному и тому же шаблону. А потому, если есть некий шаблон - мы можем работать с такими данными в текстовом выводе. 

Регулярные выражения бывают очень полезны при просмотре логов, если вы хотите достать данные по определённому паттерну.

## Важные обозначения и горячие клавиши

Необходимо не просто ознакомиться с этими обозначениями, а именно ЗАПОМНИТЬ - запишите их, если необходимо, и держите в голове.

## **~** (тильда)

Домашняя директория того пользователя, под которым вы зашли в систему

В **Windows** у меня эта директория `C:\Users\Dmitrii` - т.е., я зашёл в систему под пользователем **Dmitrii**.

На **MacOS** это у меня `/Users/dmitrii`.

Пути различаются между этими двумя операционными системами, так как у них разная файловая система.

## . (точка)

Текущая директория

Например, если вы зайдёте в свою домашнюю директорию (**~**), то `.` будет показывать именно на это место (домашнюю директорию).

Зачем это нужно? - Ну например, вы захотите из какой-то другой директории скопировать какой-то файл "сюда" - в этом случае, при помощи "точки" вы сможете сказать "сюда".

## .. (две точки)

Директория на уровень выше

Если мы находимся в директории `C:\Users\Dmitrii`, то  `..` б удет указывать на `C:\Users` соответственно, ибо эта директория "выше", то есть - директория **Dmitrii** вложена в директорию **Users**.

![](https://ucarecdn.com/b1f1ea33-e39a-4c02-85f4-6954db8d6a39/)

Директория **Users**, в которую вложена директория **Dmitrii**

Знакомая картина, когда "хакеры" в кино быстро выполняют какие-то действия на компьютере?

Во многом, скорость и удобство работы с терминалом достигается за счёт горячих клавиш.

# Перемещение курсора в строке

Допустим, у вас есть достаточно длинная команда, которая как-то логически поделена (название команды, название параметров, значение параметров):

```bash
npx mocha --spec=tests/*.spec.js --timeout=10000 --file=config/setup.js
```

 Что, если вы опечатались в самом начале строки? Или в конце? Самое простое решение - это зажать на клавиатуре стрелку "влево" или "вправо" и ждать, пока курсор дойдёт до нужного места.

Однако, если делать "быстро" и "по-умному" - вы можете...

## Переключиться в начало или в конец строки

В начало строки:

- На **Windows**: клавиша `Home`,
- На **MacOS**: комбинация клавиш `Control` и `A`.

В конец строки:

- На **Windows**: клавиша `End`,
- На **MacOS**: комбинация клавиш `Control` и `E`.

## Перескочить через слово

Через слово назад:

- На **Windows**: комбинация клавиш `Control` и ⬅️ (стрелка влево),
- На **MacOS**: комбинация клавиш `Option` и ⬅️ (стрелка влево).

Через слово вперёд:

- На **Windows**: комбинация клавиш `Control` и ➡️ (стрелка вправо),
- На **MacOS**: комбинация клавиш `Option` и ➡️ (стрелка вправо).

# Очистить текущий ввод

Бывает, что ввёл команду (порой довольно длинную), и внезапно понял, что ошибся - нужна абсолютно другая команда. И вновь, самый простой выход - зажать кнопку `Backspace` (delete) и ждать, пока по одному символу удалится вся строка...

"Умный" и "быстрый" же выход - это:

- На **Windows** и **MacOS**: комбинация клавиш `Control` и `U`.

# История команд

Вы ранее вводили какую-то команду (да хоть только что), и вот она вам понадобилась вновь... И что круто - вам не нужно заново её вводить с нуля, чтобы выполнить ещё раз!

И здесь всё предельно просто - нажимайте на клавиатуре стрелки вверх/вниз (⬆️/), чтобы идти по истории команд назад/вперёд.

# Остановить выполнение команды

Если вы начали выполнять какую-то команду, и в процессе решили отказаться от её выполнения (или она зависла) - можно её принудительно остановить, зажав комбинацию клавиш `Control` и `C`. В некоторых случаях, комбинацию требуется нажать несколько раз.

# dirs

Посмотреть путь до текущей директории

## Применение:

```bash
dirs
```

При выполнении данной команды, мы увидим путь до текущей директории. Если где-то в этом пути присутствует домашняя директория, то путь будет показан **ОТ** неё.

Например, если мы в `C:\Users\Dmitrii\Music`, то dirs выведет нам `~/Music`, ибо **Dmitrii** - домашняя директория.

![](https://ucarecdn.com/93d5e6e6-9804-4898-bc34-3311a580f029/)

Чтобы увидеть полный путь - можно использовать модификатор `-l`:

```bash
dirs -l
```

![](https://ucarecdn.com/da39ec0f-26a1-471d-b99d-5d6af61281b1/)

# ls

Посмотреть файлы и папки в текущей директории

## Применение:

```bash
ls
```

![](https://ucarecdn.com/7d28d6b6-1083-485f-bb6f-e5faf3ffebb8/)

Вывод команды **ls**

## Передвижение по директориям

# cd

Перейти в директорию...

## Применение

```bash
cd <путь>
```

 Если мы находимся в директории `~` (домашней) и хотим перейти на уровень выше, то выполним:

```bash
cd ..
```

Таким образом, мы сказали "перейди в директорию уровнем выше" (в ту, в которую вложена текущая директория), т.е. - на примере в **Windows**, мы перейдём из `C:\Users\Dmitrii` в `C:\Users`.

![](https://ucarecdn.com/e3bdf404-56dc-485e-be5d-b8e907ebb73b/)

Перешли на уровень выше (в **Users**)

![](https://ucarecdn.com/b1f1ea33-e39a-4c02-85f4-6954db8d6a39/)

Директория **Users**, в которую вложена директория **Dmitrii**

А если теперь я захочу обратно перейти в директорию **Dmitrii** - я выполню:

```bash
cd Dmitrii
```

![](https://ucarecdn.com/679602ad-de11-444e-b3c4-8225fac460d7/)

Вернулись в домашнюю директорию

Однако, переходить можно не по одной директории каждый раз, а указать некую последовательность...

То есть, мы можем прийти в корень диска `C:\`, если из домашней директории `~` (или `C:\Users\Dmitrii`) выполним:

```bash
cd ../..
```

Если мы хотим по-быстрому вернуться в **домашнюю директорию** (**~**) из этого места (`C:\`), то не обязательно писать путь `Users/Dmitrii` - можно использовать **тильду**!

## Создание директории

# mkdir

Создать директорию

## Применение

```bash
mkdir <название директории>
```

Для того, чтобы создать папку (например, `Penguins`) в месте, где мы на данный момент находимся - необходимо просто выполнить:


## Копирование файлов

# cp

Копирование файлов и директорий

## Применение

```php-template
cp <путь до копируемого файла или директории> <путь до копированного файла или директории>
```

Иными словами, первый аргумент - это **путь до того, что мы хотим скопировать**, а второй аргумент - **путь до места, где мы хотим оставить копию**.

То есть, если мы хотим создать копию файла `text2.txt` и назвать её `text3.txt` - это будет выглядеть так:

```arduino
cp text2.txt text3.txt
```

![Сделали копию text2.txt и назвали её text3.txt](https://ucarecdn.com/eee375a2-b61b-48a3-ab04-9dfb123526dd/)

Сделали копию **text2.txt** и назвали её **text3.txt**

Синтаксис команды будет точно такой же, если мы хотим скопировать директорию, кроме одного момента - необходимо добавить параметр `-r`, чтобы скопировались вложенные файлы/папки:

```bash
cp -r Projects Projects-Copy
```

## Взять только необходимое из файла

# grep

Вывести только необходимый текст из файла

## Применение

`grep`, как правило, вызывается в паре с какой-либо другой командой, которая выводит текст в терминал, например - `cat`:

```php-template
cat <название файла> | grep <текст или регулярное выражение>
```

 Например, если мы имеем файл `info.txt` с содержимым:

```makefile
Name: John Doe
Email: j.doe@example.com
Address: Test Street 90210
```

Мы можем вытащить строку с адресом электронной почты по ключевому слову (Email):

```nginx
cat info.txt | grep "Email"
```

Таким образом, результат:

А если мы хотим избавиться от лишнего Email:, и хотим получить исключительно адрес электронной почты / ссылку, или что-либо ещё, что попадает под некий паттерн - можно использовать регулярное выражение.

Для **email**`ов можно использовать регулярку `\S+@\S+\.\S+`, но также придётся добавить параметры `-Eo`, чтобы команде было понятно, что теперь мы используем регулярку, а не обычный текст:

```bash
cat text.txt | grep -Eo "\S+@\S+\.\S+"
```

## Примечание

На всякий случай оставлю тут регулярку для ссылок (URL): `(http|https)://[a-zA-Z0-9./?=_%:-]*`

## Вывести последние n- строк из файла

# tail

Вывести последние n- строк из файла

## Применение

```bash
tail <название файла>
```

Данный функционал крайне полезен, если вы хотите прочитать файл с логами (который, как правило, очень большой, и самые последние логи падают именно в конец файла):

```bash
tail log.txt
```

По умолчанию данная команда выводит последние 10 строк, но это можно исправить параметром `-n <количество строк>`: 

```bash
tail -n 15 log.txt
```

## Примечание

Если вы хотите, чтобы последние **n-** строк регулярно обновлялись у вас в терминале (без необходимости руками перезапускать команду) - можно автоматизировать перезапуск **tail**`а при помощи пакета [**watch**](https://linux.die.net/man/1/watch) (но это уже другая история; данный пакет не входит в состав ОС **Windows** / **MacOS**).
